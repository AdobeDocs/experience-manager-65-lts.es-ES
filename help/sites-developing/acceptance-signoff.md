# Aceptación y aprobación{#acceptance-and-sign-off}

Como se ha mencionado anteriormente, [las pruebas de aceptación](/help/sites-developing/planning.md) son una categoría especial de pruebas que se utiliza para comprobar lo siguiente:

* El proyecto cumple los requisitos del cliente.
* El cliente acepta el proyecto.

El cliente y el administrador del proyecto deben definir los requisitos. Cada una de ellas tiene una importancia particular, ya que a menudo están relacionadas con la facturación final.

Por este motivo, los requisitos deben ser:

* Definido lo antes posible.
* Se actualiza siempre que es necesario.
* Tanto el socio como el cliente deben aceptar y firmar formalmente cualquier cambio.
